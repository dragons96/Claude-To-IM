# 环境初始化改进说明

## 改进内容

将启动脚本中的环境检查逻辑移到 Python 代码中，实现更智能的初始化。

## 之前的流程

### 启动脚本（start.sh / start.bat）

1. 检查虚拟环境是否存在
2. 如果不存在，提示用户手动运行 `uv venv`
3. 激活虚拟环境
4. 检查 .env 文件是否存在
5. 如果不存在，显示示例并询问是否继续
6. 创建必要的目录（sessions、logs）
7. 启动 Python 服务

### 问题

- ❌ 虚拟环境不存在时需要手动创建
- ❌ 检查逻辑分散在 shell 脚本中
- ❌ Windows 和 Linux 脚本逻辑重复
- ❌ 不够灵活，难以扩展

## 现在的流程

### Python 初始化（src/cli/main.py）

在启动服务之前，`check_and_initialize()` 函数会自动检查并初始化：

1. **虚拟环境检查**
   - 如果不存在，自动运行 `uv venv` 创建
   - 显示创建进度和结果

2. **.env 文件检查**
   - 如果不存在，显示完整的配置示例
   - 询问用户是否继续启动

3. **创建必要目录**
   - `sessions/` - 会话数据目录
   - `logs/` - 日志目录
   - `sessions/database/` - 数据库目录

4. **启动服务**

### 简化的启动脚本

启动脚本现在只负责：
1. 激活虚拟环境（如果存在）
2. 启动 Python 服务

所有环境检查都在 Python 代码中完成。

## 优势

✅ **自动化**：虚拟环境不存在时自动创建
✅ **跨平台**：检查逻辑用 Python 实现，Windows/Linux 通用
✅ **易维护**：检查逻辑集中在一处
✅ **更友好**：清晰的消息提示和进度显示
✅ **可扩展**：易于添加新的检查项

## 使用方法

### 方法1：使用启动脚本（推荐）

**Linux/Mac:**
```bash
./bin/start.sh
```

**Windows:**
```cmd
bin\start.bat
```

### 方法2：直接运行 Python

```bash
# 激活虚拟环境
source .venv/bin/activate  # Linux/Mac
# 或
.venv\Scripts\activate.bat  # Windows

# 运行服务（会自动进行环境检查）
python -m src.cli.main
```

## 配置检查项

### 必需项

- ✅ 虚拟环境（.venv）
- ✅ .env 配置文件
- ✅ sessions/ 目录
