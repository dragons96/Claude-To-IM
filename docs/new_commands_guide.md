# 新命令使用指南

本文档介绍了两个新增的命令：`/delete` 和 `/session:exec`。

## `/delete` - 删除会话命令

### 功能说明
删除指定的 Claude 会话及其所有关联的消息历史记录。

### 语法
```
/delete <会话ID>
```

### 参数
- `会话ID`: 要删除的会话ID（数据库中的id字段）

### 使用示例
```
/delete abc123-def456-ghi789
```

### 注意事项
- 删除操作不可逆，删除后无法恢复
- 会删除该会话的所有消息历史记录
- 删除后会尝试关闭 Claude SDK 会话
- 如果删除的是当前活跃会话，下次发送消息时会自动创建新会话

### 返回结果
成功时返回：
```
✅ 成功删除会话

📋 已删除会话信息:
• 会话ID: abc123-def456-ghi789
• SDK Session ID: sdk-session-xyz
• 工作目录: /path/to/session

💡 提示:
• 使用 /sessions 查看剩余会话
• 使用 /new 创建新会话
```

失败时返回相应的错误提示。

## `/session:exec` - 在指定会话执行命令

### 功能说明
在不切换当前活跃会话的情况下，向指定的会话发送消息并执行。

### 使用场景
- **多会话并行管理**: 同时在多个项目或任务之间工作
- **不中断当前工作**: 在保持当前会话活跃的同时，在其他会话中执行任务
- **批量操作**: 依次在多个会话中执行相同的操作

### 语法
```
/session:exec <会话ID> <内容>
```

### 参数
- `会话ID`: 目标会话的ID（通过 /sessions 查看）
- `内容`: 要发送到目标会话的消息内容

### 使用示例

#### 示例1: 在不同会话中分析代码
```
# 当前在项目A的会话中
/session:exec abc123 运行测试用例

# 继续在项目A工作，同时在项目B会话中执行命令
/session:exec def456 构建项目

# 在项目C会话中分析代码
/session:exec ghi789 分析 main.py 的性能问题
```

#### 示例2: 多项目并行管理
```
# 查看所有会话
/sessions

# 在项目A会话中运行测试
/session:exec session-a 运行所有单元测试

# 在项目B会话中构建
/session:exec session-b 执行 npm run build

# 在项目C会话中部署
/session:exec session-c 部署到生产环境
```

### 返回结果
命令执行后，Claude 的响应会返回到当前的飞书会话中，但实际执行是在指定的 Claude 会话中。

### 注意事项
- **不会切换活跃会话**: 当前活跃会话保持不变
- **响应返回到当前会话**: 目标会话的响应会显示在当前的飞书对话中
- **会话必须存在**: 目标会话必须已经创建，否则会报错
- **权限验证**: 只能操作属于当前平台会话的 Claude 会话

### 工作流程
1. 用户发送 `/session:exec <会话ID> <内容>`
2. 系统验证目标会话是否存在
3. 将消息内容发送到指定的 Claude 会话
4. 获取 Claude 的响应
5. 将响应返回到当前的飞书会话

### 与 `/switch` 的区别

| 特性 | `/switch` | `/session:exec` |
|------|-----------|-----------------|
| 切换活跃会话 | ✅ 是 | ❌ 否 |
| 后续消息去向 | 切换后的会话 | 仍然发送到原活跃会话 |
| 适用场景 | 长期工作在某会话 | 临时在其他会话执行任务 |
| 命令次数 | 需要切换两次 | 一次执行 |
| 响应显示 | 正常显示 | 显示在当前飞书会话 |

### 推荐使用场景

#### 使用 `/switch` 的场景
- 长期在一个项目中工作
- 需要连续对话和上下文
- 频繁在同一个会话中交互

#### 使用 `/session:exec` 的场景
- 临时在其他会话执行单个任务
- 不想中断当前工作流
- 需要在多个会话之间快速切换执行命令
- 批量管理多个项目

## 常见问题

### Q: 删除会话后，工作目录的文件会被删除吗？
A: 不会。`/delete` 命令只删除数据库中的会话记录和消息历史，不会删除工作目录中的文件。如需清理文件，请手动删除。

### Q: 如何获取会话ID？
A: 使用 `/sessions` 命令查看所有会话，会显示每个会话的ID。

### Q: `/session:exec` 可以在群聊中使用吗？
A: 可以。`/session:exec` 在私聊和群聊中都可以使用，但需要注意会话必须属于当前的飞书会话。

### Q: 如果目标会话不存在怎么办？
A: 命令会返回错误提示，提示你使用 `/sessions` 查看可用的会话。

### Q: 可以同时执行多个 `/session:exec` 命令吗？
A: 可以。你可以连续发送多个 `/session:exec` 命令，它们会按顺序依次执行。

## 技术实现细节

### `/delete` 命令实现
1. 验证平台会话存在
2. 验证目标 Claude 会话存在且属于当前平台会话
3. 尝试关闭 Claude SDK 会话
4. 删除数据库中的消息历史记录
5. 删除数据库中的会话记录

### `/session:exec` 命令实现
1. 解析参数：会话ID + 消息内容
2. 验证目标会话存在
3. 构造新的消息对象（替换内容）
4. 返回特殊标记 `{"type": "exec_in_session", ...}`
5. Adapter 接收特殊标记，调用 `route_to_claude_with_session`
6. 在指定的会话中执行消息
